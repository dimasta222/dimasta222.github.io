# Конструктор футболок

Этот файл описывает текущую структуру конструктора и назначение файлов, которые были вынесены из App.jsx.

## Общая схема

Сейчас конструктор разделен на несколько уровней:

- App.jsx
  - подключает страницу конструктора
  - подготавливает products
  - передает странице только данные конструктора и navigation callback
- ConstructorPage.jsx
  - page-layer конструктора
  - связывает hook состояния с UI-компонентами
  - собирает layout страницы конструктора
- useConstructorState.js
  - хранит все состояние конструктора
  - собирает derived-данные для превью и блока заказа
  - содержит handlers для выбора товара, цвета, загрузки макета, фигур и drag-and-drop
- shared textileHelpers.js
  - хранит общие textile-helper'ы
  - используется конструктором и остальной textile-логикой
- shared textilePreviewHelpers.js
  - хранит preview/gallery helper-кластер для футболок
  - используется конструктором и textile-превью
- constructorConfig.js
  - хранит constructor-specific конфиг и утилиты
  - хранит shape-категории и набор фигур для отдельного слоя «Фигуры»
  - собирает список товаров конструктора
  - формирует Telegram-ссылку заказа
- UI-компоненты в папке constructor
  - каждый отвечает только за свой визуальный блок

## Файлы и ответственность

### App.jsx

Роль:

- точка входа страницы конструктора на уровне приложения
- готовит constructor products
- передает в ConstructorPage только данные конструктора

Что осталось здесь по конструктору:

- создание CONSTRUCTOR_PRODUCTS
- подключение ConstructorPage
- передача styles, logo-компонента и navigation callback

### components/constructor/ConstructorPage.jsx

Роль:

- отдельный page-layer для конструктора
- связывает useConstructorState.js с presentational-компонентами
- держит layout страницы вне App.jsx

Что находится здесь:

- вызов useConstructorState(...)
- buildPreviewSrc для mockup-превью
- сборка layout из tabs, sidebar, preview и order panel
- подключение onBack, LogoMini и общих app styles

Идея:

- App.jsx остается orchestration-слоем приложения
- constructor page orchestration живет рядом с constructor UI и сам импортирует свои общие зависимости

### hooks/useConstructorState.js

Роль:

- основной state layer конструктора
- единое место для constructor-state, derived values и handlers

Что хранится внутри:

- активный таб
- выбранный текстиль
- сторона печати
- цвет
- размер
- количество
- массив слоёв
- активный слой
- порядок слоёв
- видимость и блокировка слоёв
- масштаб и позиция активного слоя
- состояние drag-and-drop для активного слоя
- шрифт и семейство для текстового слоя
- shape layer с отдельным key фигуры, основной заливкой, контуром, масштабом и shape-эффектами
- text box модель текста с автопереносом по ширине контейнера и resize через handles в превью
- межстрочный интервал, обводка и режимы тени текстового слоя
- межбуквенный интервал и выравнивание текстового слоя

Что считает внутри:

- текущий product
- print area
- preview source
- текущую цену
- order meta
- telegram link
- сводку по слоям для заказа

Что обрабатывает внутри:

- смену товара
- смену цвета
- создание нового текстового слоя с заметной primary CTA-кнопкой во вкладке текста
- создание нового текстового слоя с пустым содержимым и немедленным показом на превью
- переключение между текстовыми слоями кликом по карточке слоя во вкладке текста или прямо по тексту в превью
- прямое редактирование текста прямо в слое на превью вместо отдельного textarea в левой панели
- быстрое скрытие и повторное отображение отдельных текстовых слоёв прямо во вкладке текста для удобного редактирования перекрывающихся слоёв
- быстрое удаление отдельных текстовых слоёв прямо во вкладке текста
- менеджер слоёв с одиночным кликом для простого выделения активного слоя и двойным кликом для перехода в его раздел редактирования
- отдельные наборы слоёв для сторон «Спереди» и «Сзади»: каждый новый текст, shape-layer или добавленный на превью макет привязывается к текущей стороне и не переносится на другую
- реальные front/back PNG-мокапы для чёрной, белой, розовой, бежевой и серой oversize-модели с калиброванной print-area и fallback на SVG-мокапы для остальных размеров, цветов и моделей
- размерозависимые oversize-mockup override-ы: выбранный размер может переопределять реальный PNG-мокап и параметры зоны печати отдельно для front/back, а если для размера ассет ещё не добавлен, конструктор остаётся на SVG-fallback без поломки превью
- смена размера футболки теперь не только меняет подпись максимальной зоны, но и пересчитывает физические лимиты для текста, фигур, линий и загруженных картинок; существующие upload/shape/line слои автоматически подгоняются под новую зону выбранного размера
- разное стартовое смещение для новых текстовых, shape- и upload-слоёв, чтобы добавление нового элемента было заметно сразу на превью; новый текст стартует немного выше геометрического центра print-area
- новый text-layer создаётся со стартовой шириной textBox 60% от ширины print-area
- компактный менеджер слоёв: слева drag-handle, по центру реальное содержимое слоя, справа постоянные кнопки скрытия и удаления без лишних служебных меток
- ручную перестановку слоёв перетаскиванием карточек вверх и вниз прямо во вкладке «Слои»
- быстрые кнопки скрытия и удаления прямо у карточек слоя во вкладке «Слои»
- sticky toolbar быстрых текстовых действий в отдельном верхнем слоте над превью с выбором шрифта, отдельной palette-кнопкой цвета, размером, выравниванием и переключением панелей «Интервалы»/«Эффекты»; при выборе text-layer на превью toolbar показывается вне зависимости от активной вкладки, а его панели временно открываются в левой колонке поверх текущей вкладки с закрытием по крестику или клику в пустое место превью
- центральная колонка превью больше не использует sticky-позиционирование: сам мокап и верхние кнопки не должны сдвигаться вверх или вниз при появлении text/shape toolbar, а двигаются только вместе со скроллом страницы
- выбор цвета текста через три секции: свободный picker + HEX, готовые сплошные цвета и дефолтные градиенты
- создание нового слоя-фигуры через каталог фигур по умолчанию и отдельный режим замены фигуры в уже выбранном shape-layer
- создание нового слоя-фигуры с сохранением реальных пропорций SVG на превью, а не через квадратный стартовый бокс
- показ физического размера для фигур из категории «Основные фигуры» в sidebar и summary по intrinsic-пропорциям самой SVG: круг, квадрат и базовые треугольники не искажают пользовательскую метрику, даже если preview отдельно компенсирует искажение mockup
- выбор шрифта для текстового слоя с локальным поиском, исправлением ошибочной раскладки, клавиатурной навигацией, очисткой запроса, scrollable listbox-списком, автопрокруткой к активному результату и закреплением активного шрифта сверху
- показ только одной активной панели настроек под текстовыми слоями вместо длинного непрерывного списка
- показ короткого фрагмента текста у текстовых слоёв в менеджере слоёв и в списке текстовых слоёв, а до ввода текста сохранение fallback-подписи вида «Текст N»
- resize активного слоя прямо на превью: для текста боковые маркеры меняют ширину контейнера и перенос, угловые масштабируют контейнер и размер текста как единый объект, для upload/shape угловой resize остаётся пропорциональным с опорой на противоположный угол, а боковые маркеры фиксируют противоположную сторону; у shape-layer при этом деформируется сама фигура вместе с рамкой
- mouse marquee-selection на всём превью: можно протянуть рамку по пустому месту, выделить сразу несколько объектов текущей стороны и затем удалить их через Backspace/Delete
- расширенный каталог «Основные фигуры»: кроме базовых квадрата, круга и треугольников в нём есть плюс, параллелограммы, трапеции, арка, щит, полукруг, четверть круга и дуги
- настройку ширины text box и автопереноса через resize на превью
- snapping при drag и resize к краям, центру print-area и к фактическим границам других видимых слоёв с отображением smart guides в превью
- показ только фактического размера текста в сантиметрах в sidebar на основе реального DOM-измерения внутри physical print-area
- ручной ввод ширины и высоты активного text-layer в сантиметрах в sidebar с жёстко связанными пропорциями: изменение одного поля автоматически масштабирует другое, размер текста, textBoxWidth, межбуквенный интервал, контур, обводку и тень через `scaleActiveTextLayer` множителем `newCm / currentCm`
- фактические размеры текста и фигур в sidebar учитывают не только базовую геометрию, но и визуальное расширение от text stroke, text shadow и shape effects; для обычных фигур внутренняя обводка не увеличивает внешний фактический размер, а для фигур расчет идет в физических логических единицах конструктора, поэтому квадратные и пропорциональные формы не искажают соотношение сторон из-за геометрии мокап-превью
- сохранение пустого text-layer видимым в превью как рабочего text box с placeholder
- удаление активного слоя через кнопку прямо на превью и через клавиши Backspace/Delete вне полей ввода
- настройку межстрочного интервала, свободного цвета обводки и мягкой тени текста
- настройку межбуквенного интервала и выравнивания текста
- настройку фигуры через sticky shape-toolbar в отдельном верхнем слоте над превью: кнопку «Редактировать» для замены фигуры в текущем слое с подсветкой только в active replace-режиме и автоматическим переходом на вкладку «Фигуры», показ toolbar при выборе фигуры на превью даже вне вкладки «Фигуры», временное открытие левой shape-панели поверх текущей вкладки по кнопкам «Цвет» / «Цвет обводки» / «Эффекты» с закрытием по крестику или клику в пустое место превью, выбор основного цвета, quick-popover «Обводка», дополнительный popover «Углы» для фигур с острыми углами и отдельную левую панель цвета обводки и панель эффектов
- параметр `cornerRoundness` для всех фигур каталога с жёсткими polygon-углами: квадрат, многоугольники, звёзды, стрелки, баннеры и другие угловые shape-формы могут сглаживать углы через toolbar-поповер вплоть до максимально округлённого состояния
- выход из режима замены фигуры по клику на другой слой или на пустое место превью с возвратом каталога в обычный режим добавления
- хранение размеров upload и обычных shape-layer в сантиметрах относительно физической зоны печати 40 × 50 см, а для line-shape хранение длины и технической высоты в логических px холста с пересчётом фактического размера в сантиметры для sidebar и summary
- новый каталог фигур в левой колонке: вертикальный список категорий, горизонтальные ленты превью, кнопка «Показать все» и отдельный экран выбранной категории без текстового поиска
- настройку эффектов фигуры через взаимоисключающие режимы «Падающая тень» и «Искажение» с направлением, смещением и отдельными цветами
- внутреннюю обводку фигуры по контуру, включая сплошной, пунктирный и точечный режимы, завязанные на толщину линии
- рамку shape-layer с учётом эффектов: preview-box расширяется под тень, искажение и обводку и остаётся вплотную к реальной фигуре без обрезания эффекта
- отдельный line-special-case: линейные фигуры изначально добавляются заметнее, живут в логической пиксельной системе холста, сразу активируют в popover «Обводка» тот режим линии, который задан у выбранного line-shape через `defaultLineStyle`, меняют длину и угол через перетаскивание левого или правого endpoint-handle с фиксацией противоположного конца, держат геометрический минимум длины от текущей толщины и типа концов, держат толщину как отдельный `strokeWidth`-параметр, симметрично утолщают концевые стрелки и декоративные точки/квадраты/ромбы, регулируют толщину через «Обводка» → «Толщина» в диапазоне 1–100 px и не показывают size-slider в левой панели
- factual size активной фигуры в sidebar для обычных shape-layer теперь отделяет intrinsic width-derived состояние от ручной деформации: симметричные фигуры на старте снова показывают симметричный размер, а после вертикального растяжения всё равно не выходят за физический лимит выбранной print-area
- factual size текста в sidebar учитывает stroke, shadow и underline, но итоговая ширина и высота clamp-ятся к физическим пределам текущей print-area и не могут превышать максимальную зону печати выбранного размера
- при смене размера текстиля factual size текста пересчитывается заново даже если пиксельная рамка текста визуально не изменилась: кэш текстовых метрик теперь учитывает физические размеры текущей print-area
- upload-layer при добавлении и дальнейших resize снова сохраняет исходную пропорцию загруженного файла: стартовые размеры берутся из реального bitmap aspect ratio, а рендер на превью использует `object-fit: contain` без компенсационного растяжения
- side-handle resize у upload-layer тоже снова масштабирует файл пропорционально: при перетаскивании левой/правой или верхней/нижней стороны меняются обе оси изображения, а не только контейнер
- загрузку файла в постоянную библиотеку миниатюр во вкладке Upload с ручным добавлением слоя на превью и отдельным ручным удалением файла из списка
- side-aware summary заказа: отдельный учёт видимых слоёв спереди и сзади в orderMeta и в Telegram-сообщении
- расчёт стоимости заказа поштучно: к базовой цене футболки добавляется цена печати по одной или двум сторонам, а блок заказа показывает итог как `Печать односторонняя` или `Печать двусторонняя`; формат каждой стороны подбирается по общему занимаемому bounding box всех видимых слоёв через `getConstructorPrintFormat(...)`
- чтение размеров загруженного изображения
- изменение физического размера активного upload/shape слоя в сантиметрах
- удаление слоя
- дублирование слоя
- смену порядка слоёв
- переключение видимости и блокировки
- pointer drag активного слоя
- возврат активного слоя в центр

Идея:

- UI-компоненты не должны знать внутреннюю механику конструктора
- они получают уже готовые props и вызывают callbacks

### shared/textileHelpers.js

Роль:

- общий доменный модуль текстиля
- место для reusable textile-helper'ов, которые не должны жить внутри App.jsx

Что находится здесь:

- parseColorOptions(...)
- getDefaultTshirtColor(...)
- resolveColorSwatch(...)
- normalizeColorName(...)
- normalizeVariantLabel(...)
- getTshirtSizes(...)
- parsePriceValue(...)

Идея:

- конструктор и textile-страницы используют один общий слой helper-функций
- это снижает связность App.jsx и делает границы ответственности понятнее

### shared/textilePreviewHelpers.js

Роль:

- общий модуль textile preview/gallery helper'ов
- место для SVG preview builder и gallery asset logic, которые не должны жить в App.jsx

Что находится здесь:

- svgToDataUri(...)
- buildOrderedGalleryCandidates(...)
- resolveHomepageTshirtPreview(...)
- preloadHomepageTshirtPreview(...)
- loadImageCandidate(...)
- buildTshirtMockupSvg(...)
- buildHomepageTshirtPlaceholderSvg(...)
- buildTshirtFallbackSlides(...)

Идея:

- preview и gallery helper'ы образуют отдельный shared-кластер
- constructor preview builder теперь зависит от shared preview module, а не от локальных функций в App.jsx

### components/constructor/constructorConfig.js

Роль:

- модуль constructor-specific конфигурации и утилит

Что находится здесь:

- CONSTRUCTOR_PRINT_AREAS
  - зоны печати для разных моделей футболок
- CONSTRUCTOR_TABS
  - список табов левой панели, включая вкладки «Слои» и «Фигуры»
- buildConstructorProducts(...)
  - собирает плоский массив товаров конструктора из каталога текстиля
- buildConstructorTelegramLink(...)
  - формирует сообщение и ссылку для Telegram с перечислением всех слоёв, формата печати, занимаемой области и цены печати по стороне
- CONSTRUCTOR_TEXT_FONTS и getConstructorTextFont(...)
  - задают набор предустановленных шрифтов для текстовых слоёв, который используется в поиске, группировке и выборе шрифта
- CONSTRUCTOR_TEXT_SOLID_COLORS, CONSTRUCTOR_TEXT_GRADIENTS, CONSTRUCTOR_SHAPE_BASIC_COLORS и getConstructorTextGradient(...)
  - задают палитры цветов и градиентов для текста, фигур и shape-эффектов
- CONSTRUCTOR_SHAPE_CATEGORIES, CONSTRUCTOR_SHAPES, getConstructorShape(...) и buildConstructorShapeSvg(...)
  - задают расширенный каталог категорий фигур, shape-definitions и SVG-сборку основной фигуры, внутренней обводки и её цветовых копий для эффектов
- readFileAsDataUrl(...)
  - читает загруженный файл как data URL для библиотеки upload-файлов
- readImageSize(...)
  - получает размеры изображения

Идея:

- все, что относится именно к конструктору как к модулю, лучше держать здесь
- это упрощает поддержку отдельного page-layer конструктора

### components/constructor/ConstructorTabsNav.jsx

Роль:

- верхняя навигация по табам конструктора

Что делает:

- показывает список вкладок
- отображает активную вкладку
- вызывает onTabChange при переключении

Что не делает:

- не хранит состояние
- не знает ничего про товар, цену, файл, текст или порядок слоёв

### components/constructor/ConstructorSidebarPanel.jsx

Роль:

- левая панель конструктора

Что делает:

- показывает содержимое активного таба
- рендерит блоки:
  - слои
  - текстиль
  - загрузка макета
  - текст
  - фигуры
- во вкладке Upload показывает постоянный список загруженных файлов в виде миниатюр и кнопки ручного добавления/удаления

Что получает через props:

- текущий активный таб
- список товаров
- выбранный товар
- размер, цвет, количество
- список слоёв

## Поток данных

Схема упрощенно выглядит так:

1. App.jsx собирает constructor products
2. App.jsx рендерит ConstructorPage.jsx и передает app-level зависимости
3. ConstructorPage.jsx вызывает useConstructorState(...)
4. Hook возвращает:
   - state
   - derived values
   - handlers
5. ConstructorPage.jsx раздает эти данные в:
   - ConstructorTabsNav
   - ConstructorSidebarPanel
   - ConstructorPreviewPanel
   - ConstructorOrderPanel

Идея разделения:

- состояние и логика живут в hook
- конфиг и helper-функции живут в constructorConfig.js
- визуальные части живут в отдельных UI-компонентах
- текст, фигуры и добавленные на превью макеты собраны в единую модель слоёв, а загруженные файлы до добавления слоя хранятся отдельно в upload-библиотеке

## Зачем это разделение

Что это дает:

- App.jsx становится заметно проще читать
- легче искать хвосты и мертвый код
- проще править отдельные части конструктора без риска сломать весь экран
- проще тестировать constructor page отдельно от корневого App.jsx
- проще понимать границы ответственности каждого файла

## Что еще можно вынести дальше

Следующие естественные шаги, если будем продолжать:

- развивать эффекты и дополнительные пресеты поверх текстовых слоёв
- при необходимости вынести layer helpers в отдельный constructor-модуль
